# 영어 학습 플랜

Last updated: 2026-05-15 17:20:39 KST (+0900)

## 운영 방식

Day가 아니라 Step 기준으로 진행한다.

이유:
- 현민의 시간 할당이 유동적임.
- 하루에 30분만 할 수도 있고, 1~2시간 할 수도 있음.
- 하루에 Step 1개만 할 필요 없음.
- 시간이 많으면 Step 2~3개 진행 가능.

## Step 진행 규칙

수업 방식과 의도는 별도 문서에 기록한다.

- `teaching-method.md`
  - 강의 + 문장 만들기 + 첨삭 + 실전 적용을 어떻게 섞을지 정의한다.
  - 현민이 먼저 영어로 써보고 내가 첨삭하는 방식은 초반 약점 진단과 문장 엔진 훈련을 위한 의도된 방식이다.

한 Step은 보통 다음 구성으로 만든다.

1. 핵심 문법/표현 1개
2. 필수 예문 5개
3. 현민의 업무/생활 기반 예문
4. 연습 문제 10개
5. 추가 쓰기 1개
6. 숙제 첨삭 후 약점 기록

## Step 완료 조건

아래 조건을 만족하면 다음 Step으로 넘어간다.

- 숙제 제출 완료
- 핵심 패턴에서 큰 오류 없음
- 반복 오류가 있으면 다음 Step 첫 부분에서 짧게 복습
- 현민이 해당 패턴을 인지하고, 이해하고, 직접 활용할 수 있음

원칙:
- 어설프게 알고 넘어가지 않는다.
- Step이 길어져도 괜찮다.
- 필요하면 같은 Step 안에서 예문, 변형 문제, 짧은 글쓰기, 말하기 시뮬레이션을 반복한다.
- “설명은 이해했지만 직접 못 쓰는 상태”는 완료로 보지 않는다.
- 단, 완벽주의 때문에 진도를 막지는 않는다. 목표는 100점이 아니라 “실제로 쓸 수 있는 수준”이다.

다음 Step으로 넘어갈 수 있는 기준:
- 같은 패턴의 새 문장을 스스로 3~5개 만들 수 있음.
- 이전에 틀린 유형을 다시 봤을 때 스스로 고칠 수 있음.
- 실생활/업무 상황에서 최소 1개 문장으로 적용할 수 있음.

반복 오류는 `progress.md`에 누적해서 관리한다.

## 3개월 목표

### 1개월차

기본 문장 구조 잡기:
- I am
- I do
- I did
- I want to
- I need to
- I can
- I have

### 2개월차

업무 영어 + 긴 문장 읽기:
- 개발 문서
- 에러 메시지
- API docs
- infra docs

### 3개월차

기본 회화 자동화:
- 자기소개
- 업무 설명
- 문제 설명
- 질문하기
- 의견 말하기

## 매일 30분 기본 루틴

0~5분: 이전 Step 문장 복습
- 문장 5개 소리 내서 읽기
- 한국어 보고 영어로 다시 말하기

5~15분: 핵심 문법 1개

15~25분: 실전 문장 만들기
- 실제 생활/업무 문장으로 10개 만들기

25~30분: 짧은 말하기/쓰기
- 오늘 배운 문장으로 3~5문장 말하기 또는 쓰기

## 60분 루틴

- 10분 복습
- 15분 문법
- 15분 단어/표현
- 15분 개발 문서 읽기
- 5분 요약 말하기

## 120분 루틴

- 10분 복습
- 20분 문법
- 20분 문장 만들기
- 20분 리딩
- 20분 리스닝
- 20분 회화/롤플레이
- 10분 정리
