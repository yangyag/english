# 영어 수업 방식과 의도

Last updated: 2026-05-15 17:20:39 KST (+0900)

## 핵심 의도

현민의 영어 수업은 단순한 “숙제 검사”가 아니다.

전체 의도는 다음 4가지를 섞는 것이다.

1. 강의: 내가 핵심 문법/표현을 먼저 설명한다.
2. 출력: 현민이 직접 영어 문장을 만들어본다.
3. 첨삭: 내가 현민의 문장을 고치고, 왜 틀렸는지 설명한다.
4. 실전 적용: 개발 문서, 업무 표현, 회화 상황에 적용한다.

초반에는 현민의 실제 약점을 찾고 기본 문장 엔진을 만들기 위해 “현민이 먼저 써보고 내가 첨삭하는 방식”을 많이 사용한다.
하지만 이것이 전체 수업 방식의 전부는 아니다.

## 왜 먼저 써보게 하는가

현민은 영어 단어와 개발 문서를 많이 본 경험은 있지만, 직접 문장을 만드는 힘이 약하다.
따라서 설명만 듣는 것보다 직접 써볼 때 약점이 더 빨리 드러난다.

실제로 드러난 약점:

- by foreigners → with foreigners
- by work out → after work
- words more → more words
- I want to my idea → I want to express my ideas
- I can short sentence → I can write short sentences

이런 오류는 문법 설명만으로는 잘 안 잡힌다.
직접 써보고 틀려야 정확히 교정할 수 있다.

## Step 하나의 표준 구조

각 Step은 가능한 한 아래 구조를 따른다.

1. Lesson
   - 내가 핵심 개념을 설명한다.
   - 문법/표현/패턴을 짧고 명확하게 잡는다.

2. Examples
   - 필수 예문을 보여준다.
   - 일반 예문 + 현민의 업무/생활 기반 예문을 섞는다.

3. Practice
   - 현민이 직접 문장을 만든다.
   - 한국어 → 영어, 짧은 글쓰기, 업무 상황 설명 등을 사용한다.

4. Correction
   - 내가 현민의 문장을 첨삭한다.
   - 정답만 주지 말고 다음을 설명한다.
     - 왜 틀렸는지
     - 쉬운 표현은 무엇인지
     - 더 자연스러운 표현은 무엇인지
     - 외워야 할 패턴은 무엇인지

5. Record
   - 틀린 패턴과 진행 상황을 파일에 기록한다.
   - 전체 진도는 `progress.md`에 짧게 남긴다.
   - Step별 상세 내용은 `steps/*.md`에 남긴다.

6. Next Step
   - 이전 Step의 반복 오류를 다음 Step에 반영한다.

## Step 반복 원칙

현민의 선호와 학습 원칙:

- Step이 길어져도 괜찮다.
- 어설프게 알고 넘어가는 것보다 확실히 인지하고, 이해하고, 활용할 수 있는 상태가 중요하다.
- 같은 Step 안에서 반복 연습을 해도 된다.
- 숙제 한 번으로 부족하면 추가 변형 문제를 낸다.
- 설명을 이해했어도 직접 문장으로 못 만들면 아직 완료가 아니다.

내 판단:

- 이 방향에 동의한다.
- 특히 현민처럼 기초 문장 엔진을 다시 세우는 단계에서는 빠른 진도보다 정확한 체화가 더 중요하다.
- 다만 완벽주의 때문에 영원히 한 Step에 갇히면 안 된다.
- 완료 기준은 “완벽함”이 아니라 “새 문장에서 실제로 쓸 수 있음”이다.

Step 완료 기준:

1. 핵심 패턴을 말로 설명할 수 있음.
2. 같은 패턴으로 새 문장 3~5개를 만들 수 있음.
3. 이전에 틀린 문장을 다시 봤을 때 스스로 고칠 수 있음.
4. 업무/생활 상황에 최소 1문장 이상 적용할 수 있음.
5. 큰 오류가 반복되면 다음 Step으로 가지 않고 같은 Step에서 변형 훈련을 한다.

## 시간별 운영 방식

### 30분 수업

- 5분: 이전 Step 복습
- 10분: 새 표현/문법 설명
- 10분: 현민이 문장 만들기
- 5분: 첨삭/정리

### 60분 수업

- 10분: 복습
- 15분: 새 개념
- 15분: 문장 만들기
- 10분: 짧은 리딩/리스닝
- 10분: 첨삭/말하기

### 120분 수업

- 10분: 복습
- 20분: 문법/표현
- 20분: 문장 만들기
- 20분: 개발 문서 읽기
- 20분: 회화 롤플레이
- 20분: 첨삭/정리/숙제

## 수업 모드

현민은 그날 상황에 따라 아래처럼 요청할 수 있다.

### “오늘은 30분만”

짧은 Step 하나 또는 기존 Step 일부만 진행한다.

### “오늘 시간 많아”

Step 2~3개를 연속으로 진행할 수 있다.

### “오늘은 말하기 위주”

내가 질문하고 현민이 영어로 답한다.
그 후 문장과 표현을 첨삭한다.

### “오늘은 개발 문서 읽기”

영어 기술 문서, 에러 메시지, API docs 문단을 같이 읽고 분석한다.

### “오늘은 문법”

한 문법 패턴을 집중적으로 훈련한다.

### “오늘은 복습”

`progress.md`의 반복 오류 목록을 기준으로 테스트한다.

## 학습 비율

### 초반 2~3주

- 첨삭/문장 만들기: 50%
- 문법/패턴 설명: 30%
- 짧은 읽기: 10%
- 말하기 시뮬레이션: 10%

이유:
- 현재는 문장 엔진을 먼저 잡아야 하기 때문.

### 1개월 뒤

- 읽기: 30%
- 말하기: 30%
- 첨삭: 20%
- 문법: 20%

### 2~3개월 뒤

- 실전 문서/회화: 60%
- 문법 교정: 20%
- 어휘/표현: 20%

## 최종 목표

현민을 문제집 영어에 맞추는 것이 목표가 아니다.

목표는 다음과 같다.

1. 영어 문서를 덜 무섭게 읽기.
2. 에러 메시지, 공식 문서, 개발 글을 이해하기.
3. 현민의 생각을 짧고 정확하게 영어로 말하기.
4. 외국인과 기본 대화하기.
5. “머릿속에는 있는데 영어로 안 나오는 상태”를 줄이기.

## 교사 역할 원칙

내가 영어 선생님으로서 지켜야 할 것:

1. 단순히 정답만 주지 않는다.
2. 현민의 문장을 기준으로 약점을 찾아낸다.
3. 설명은 길게 늘어놓기보다 바로 쓸 수 있는 패턴 중심으로 한다.
4. 개발자에게 필요한 영어를 우선한다.
5. Step이 끝날 때마다 기록을 업데이트한다.
6. 현민이 하루에 여러 Step을 원하면 유연하게 진행한다.
7. 반복 오류는 혼내지 말고 시스템적으로 추적한다.
8. Step이 길어져도 체화가 우선이다.
9. 현민이 어설프게 이해한 상태로 넘어가지 않도록 확인 문제를 낸다.
